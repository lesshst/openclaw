# 与上游 OpenClaw 的主要变更清单

## 对比范围

- 本清单基于当前仓库 `main` 分支的已提交内容，与上游 `openclaw/openclaw` 的 `main` 分支做对比。
- 本次比对时的共同基线提交为 `a509154be5fd3adeb8a56ec9843df1a254b9cc67`。
- 当前 fork 相对上游共有 8 个独有提交，但真正有明确功能或运维含义的改动集中在少数几个点。

## 结论摘要

1. 为 WhatsApp Web 接入链路增加了 `HTTP_PROXY` / `HTTPS_PROXY` 代理支持。
2. 新增了一套本机源码仓库使用的 gateway 启停、重启、状态检查脚本。
3. 将 gateway 运行方式改成 macOS `launchd` 常驻模式，并补充安装、卸载与健康检查能力。
4. 增加了 WhatsApp 代理补丁说明文档，并忽略了本地运行生成的 `.gateway.pid` 文件。
5. `pnpm-lock.yaml` 存在较大差异，但未看到对应的核心源码或 `package.json` 同步改造，当前更像锁文件漂移或合并副产物，不视为主要产品功能调整。

## 主要改动详解

### 1. WhatsApp Web 代理支持

- 影响文件：`src/web/session.ts`
- 对应提交：`f1ffe476411b7d09c4ac2e373ee1758503ea9ee8`

改动内容：

- 新增 `HttpsProxyAgent` 引入。
- 新增 `resolveWhatsAppProxyAgent()`，从环境变量 `HTTPS_PROXY` 或 `HTTP_PROXY` 读取代理地址。
- 在创建 Baileys socket 时，把代理同时注入到 `agent` 和 `fetchAgent`。
- 代理地址非法时安全降级，直接回退到原本的直连行为，不让进程因为代理配置错误而启动失败。

实际影响：

- 该 fork 可以在本地或服务器存在代理限制的网络环境下更稳定地连通 WhatsApp Web。
- 这属于真实功能增强，不是纯文档或运维脚本改动。

### 2. 新增本地 gateway 管理脚本

- 影响文件：
  - `scripts/gw-start.sh`
  - `scripts/gw-stop.sh`
  - `scripts/gw-restart.sh`
  - `scripts/gw-status.sh`
- 初始提交：`d944f7fd26a7ea4c231fc705875163399b008a85`

改动内容：

- 增加 gateway 启动、停止、重启、状态查看脚本。
- 状态脚本会检查：
  - `launchctl` 服务是否已加载
  - `18789` 端口是否监听
  - `http://127.0.0.1:18789/` 是否可响应
- 停止脚本会顺手清理旧的 `nohup` 启动方式残留进程。

实际影响：

- 这组脚本明显是为了本地源码目录直接运行 OpenClaw gateway 做的运维便捷化封装。
- 它们不是上游通用 CLI 的一部分，而是这个 fork 针对当前开发机环境补上的辅助能力。

### 3. 改为 macOS launchd 常驻运行

- 影响文件：
  - `scripts/gw-ha-install.sh`
  - `scripts/gw-ha-uninstall.sh`
  - `scripts/gw-start.sh`
  - `scripts/gw-stop.sh`
  - `scripts/gw-status.sh`
- 对应提交：`e1cd4bff5c29885667a88ec7ab21c0c035c3cbca`

改动内容：

- 新增 `gw-ha-install.sh`，自动生成 `~/Library/LaunchAgents/ai.openclaw.gateway-src.plist`。
- 使用 `launchctl bootstrap`、`enable`、`kickstart` 管理 gateway，而不是单纯前台运行或 `nohup`。
- `plist` 中启用了 `RunAtLoad` 和 `KeepAlive`，意味着用户登录后自动拉起，异常退出后自动重启。
- 标准输出和错误输出被写入 `/tmp/openclaw-gateway-src.log` 与 `/tmp/openclaw-gateway-src.err.log`。

本地定制特征：

- 仓库路径被写死为 `/Users/lizhibo/.openclaw/workspace/tmp-openclaw-src`。
- 端口写死为 `18789`。
- 代理地址写死为 `http://127.0.0.1:7897`。
- 服务标签写死为 `ai.openclaw.gateway-src`。

结论：

- 这不是面向所有 OpenClaw 用户的通用产品能力，而是这个 fork 针对当前 macOS 机器做的部署与常驻运行定制。

### 4. 文档与本地运行辅助改动

- 影响文件：
  - `docs/WHATSAPP_PROXY_PATCH_NOTES_2026-02-21.md`
  - `.gitignore`
- 对应提交：
  - `d86f9aada039d50183b10f2de62fbb46891d33eb`
  - `8a76d19c7c1ee051324d4357b11144eeac2fa2cd`

改动内容：

- 新增中文说明文档，记录 WhatsApp 代理补丁的背景、实现方式、使用示例与兼容性说明。
- `.gitignore` 新增 `.gateway.pid`，避免本地运行时生成的 PID 文件进入版本控制。

## 最终结论

这个 fork 相对原版 OpenClaw 的核心定制可以概括为两条主线：

1. 给 WhatsApp Web/Baileys 增加代理连通能力。
2. 给当前 macOS 本地源码环境增加一套 gateway 常驻运行与运维管理脚本。

除此之外，没有看到对 OpenClaw 核心架构、通道体系、模型调度或大面积业务逻辑的系统性重写。
